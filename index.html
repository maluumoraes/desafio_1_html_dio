O que é HTML?

O HTML é a sigla designada para HyperText Markup Language (Linguagem de Marcação de Hipertexto), uma linguagem de marcação utilizada para estruturar uma página web.

De forma didática podemos imaginar que o HTML usa de marcações (chamadas de tags) para apontar onde cada elemento se localiza dentro de uma página, quais características recebem e como interagem/aparecem ao serem acessados pelo usuário.

Tudo que estiver entre a abertura e o fechamento de uma tag é chamado de elemento.

A escrita em HTML pode ser feita em qualquer editor de texto como o bloco de notas, ou utilizar de um editor específico, como o VS Code, que permite a escrita fluída e compatível com a linguagem. E a abertura de um arquivo html será realizado em um navegador, que apresentará o conteúdo como o usuário que acessará a página web irá enxergar.

Como funcionam as marcações (tags) no HTML?

Como dito anteriormente, as tags são responsáveis por marcar a estrutura da página web.

Sua escrita é feita de um jeito especial: entre chevron, popularmente conhecidos como sinais de menor que e maior que.

A tag fica entre esses sinais, da seguinte forma:

< tag > informação < /tag >

Desse modo, de acordo com o posicionamento d é delimitado onde a informação fica dentro da página.

Além da marcação da tag, existem conceitos importantes de serem entendidos. São eles os atributos, valores e filhos.

O atributo é um termo usado dentro de uma tag que especifica seu comportamento, definindo um valor para tal.

O valor é o nome dado a tag através de um atributo. Assim, toda tag que usar aquele valor se comporta daquele modo

É importante saber que:

  • Nem toda tag aceita atributos
  • Nem todo atributo funciona em qualquer tag

Ou seja, não basta conhecer as tags e entender os atributos. É necessário saber as combinações possíveis para um código funcional.

Boas práticas na programação

Quando se trabalha com programação é necessário considerar que o código será lido e manipulado por várias pessoas ao longo do tempo.

Para isso, é necessário que qualquer um que o leia consiga compreendê-lo e manuseá-lo de forma adequada e segura.

Pensando nisso, foi proposto entre os programadores as chamadas boas práticas.

Em uma analogia, as boas práticas na programação se equiparam aos bons modos na vida em sociedade: não há uma lei dizendo “cumprimente o coleguinha”, mas fazer uso de “por favor”, “obrigada”, “com licença”, “bom dia/tarde/noite” tornam o diálogo mais harmônico.

As boas práticas na programação vão além do uso adequado dos termos, é pensar o código de forma clara. Algumas dicas para quem está iniciando na programação (como eu) são:


  • Ser claro ao escrever cada código.
  • Respeitar o espaço entre elementos, indentação. É como respeitar o espaço entre palavras, o pular linha e o parágrafo na hora de escrever uma redação
  • Usar comentários explicativos curtos e objetivos quando necessário.
  • Manter o código separado em partes afins. Como temas de um texto separado em parágrafos.

Existem outras boas práticas relacionadas a segurança, teste, versionamento e documentação, porém são temas para outro momento.
